# 🌾 RiceLodge Studio ― 作物倒伏 ビジュアルシミュレーター

直感的なUIで、稲（作物）の **倒伏（lodging）** をリアルタイムに再現・分析できる
シミュレーションアプリです。品種特性・栽培管理・気象条件を自由に変えながら、
圃場がどのように倒伏していくかを視覚的に観察し、データとして書き出せます。

旧システム（Processing による研究用バッチ実験 `s1v2`）を、
**ブラウザだけで動く対話型アプリ**として再構築しました。インストール不要・オフライン動作。

---

## 1. 使い方（クイックスタート）

1. `app/index.html` を **ダブルクリック** してブラウザで開くだけ。
   （サーバー構築・ビルド・ネット接続は不要）
2. 右上の **▶ 再生** を押すとシミュレーション開始。
3. 左の **条件設定** スライダーを動かすと結果がリアルタイムに変化します。
4. 上部の **プリセット** から典型シナリオ（台風・多肥栽培など）を即適用できます。

| 操作 | ショートカット |
|------|----------------|
| 再生 / 一時停止 | `Space` |
| リセット | `R` |
| コマ送り | ⏭ ボタン |

## 2. 調整できる条件

### 🌱 作物の特性
| 項目 | 意味 |
|------|------|
| 茎強度 | 稈の折れにくさ（弾性・折損抵抗）。高いほど倒伏に強い |
| 稈長 | 草丈。高いほどテコが長く倒れやすい |
| 株あたり茎数 | 1株の分げつ数 |
| 株間隔 | 植付け密度（cm） |
| 生育ステージ | 出穂→完熟。進むほど穂が重く倒伏しやすい |

### 🌾 播種・植付方式
| 項目 | 意味 |
|------|------|
| 植付方式 | **規則的（田植え・整列）** … 等間隔の整列植え／**ドローン散播（ランダム）** … 種をドローンから散布した不規則な間隔の圃場を再現 |
| 散播ムラ | ドローン散播時の密度ムラの強さ。0%＝ほぼ一様、100%＝塊状に密集し裸地（欠株）もできる |

> ドローン散播では平均の株密度は整列植えと揃えたうえで、**間隔だけが不規則**になります。
> 密集した塊は隣株の「寄りかかり」で連鎖倒伏しやすく、まばらな部分は単独で耐える──
> といった、散播ならではの倒伏の出方を観察できます。

### 🧪 栽培管理
| 項目 | 意味 |
|------|------|
| 窒素施肥 | 多いほど徒長・軟弱化し倒伏リスク増 |
| 土壌水分 | 高いほど根の支持力が低下（根倒伏） |

### 🌬 気象条件
| 項目 | 意味 |
|------|------|
| 平均風速 | m/s |
| 突風度 | 風速変動の大きさ（最大約1.5倍の突風） |
| 風向 | 倒れる方向 |
| 降雨量 | 穂を濡らして重くし、地盤を緩める |

### 🔗 株間の相互作用
| 項目 | 意味 |
|------|------|
| 風よけ効果 | 風上の株が風を遮る強さ。高いほど風上端から倒れ、内部は守られる |
| 寄りかかり連鎖 | 倒れた株が風下の隣株へ荷重を乗せ、倒伏が風下へ伝播する強さ |

## 3. 倒伏の物理モデル

作物倒伏の力学研究に基づく**現象論モデル**です。各茎の基部まわりの
傾斜角 θ（0°=直立, 90°=完全倒伏）を、次のモーメント収支で時間発展させます。

```
駆動モーメント  = 風荷重 Kw·v²·L·cosθ  +  自重 W·L·sinθ
復元モーメント  = 弾性剛性 kE·θ          （kE ∝ 茎強度 × 根の支持力 × 健全度）

θ が増える向き … 駆動 > 復元
θ が戻る向き   … 駆動 < 復元（無風時に立ち上がる）
θ > 45° で塑性変形 → 永久倒伏（折損、以後は立ち上がらない）
```

- 風の抗力は **風速の2乗** に比例（空気抗力）。
- 倒れるほど受風面が減り（cosθ）、一方で自重モーメントが増える（sinθ）ため、
  ある角度を超えると一気に倒れ込む“倒伏の暴走”を再現します。

### 株どうしの空間相互作用（風上→風下の伝播）
近接する株が、互いの位置関係と風向に応じて影響し合う様子を再現します。

- **風よけ（フェッチ効果）**：各株の **風上側に立っている株** が局所的に風を弱めます。
  そのため **風上端の列がまず倒れ**、内部は守られます。風上の株が倒れると遮蔽が消え、
  風が貫通して **倒伏が風下へ波のように伝播** します。
- **方向性のある寄りかかり**：倒れた株はその **倒れ込む向き（＝風下）** の隣株へ荷重を乗せます。
  風上で倒れた株だけが風下の株を押し、横風・風下側の株からは押されません（一方向の連鎖）。
- **距離減衰**：近い株ほど強く影響します（半径内を単純平均せず距離で重み付け）。

> これらは各株の「(自分→隣株)ベクトルと風向の内積の符号」で風上/風下を判定し、
> 風上の **直立株は遮蔽**・**倒伏株は寄りかかり荷重** として作用させています。
> スライダー **風よけ効果 / 寄りかかり連鎖** で強さを 0〜200% に調整でき、
> 両方を 0% にすると相互作用のない一様モデルに戻ります（比較検証用）。

> 文献的背景：倒伏は「外力＋自重による曲げモーメント」が「稈の曲げ強度」を
> 超えたときに発生し、倒伏指数は草丈・稈重に比例し折損強度に反比例します。
> 実圃場では **風上端から風下へ帯状に倒れる** のが典型で、本モデルもこれを再現します。

### モデルの妥当性（内部検証の例）
標準的な作物条件での風速応答（倒伏率）:

| 風速 | 3 | 9 | 12 | 15 | 18 | 22 m/s |
|------|---|---|----|----|----|--------|
| 倒伏率 | ~1% | ~5% | ~15% | ~46% | ~85% | ~100% |

同じ18 m/sでも品種で明確に差が出ます：強稈品種 ~15% ≪ 標準 ~90% ≪ コシヒカリ型 ~100%。

## 4. 計測・出力

- **倒伏率（45°以上）**・平均傾斜・折損率・瞬間風速をリアルタイム表示
- **倒伏度分布**（直立〜完全倒伏の5段階）バー
- **倒伏率の推移グラフ**（風速も重ね描き）
- **リスク評価バッジ**（健全 / 注意 / 警戒 / 危険）
- **CSVエクスポート**（frame, lodge_rate, broken_rate, mean_lean, wind, zone1–5）
- **画像保存**（圃場ビューのPNG）

CSVのゾーン定義は旧システム（zone1〜zone5）と互換です。

## 5. プリセット

- **天候**：晴天・微風／通常／強風注意報／台風直撃／ゲリラ豪雨＋突風
- **品種・栽培**：コシヒカリ（倒伏弱）／強稈品種／多肥栽培（徒長）／密植栽培
- **播種方式**：ドローン散播（標準ムラ）／ドローン散播（ムラ大・裸地あり）

## 7. 旧システムとの関係

`n.yuya/s1v2`（Processing）は、パラメータを少しずつ変えて多数の試行を
自動実行し CSV を蓄積する**研究用バッチツール**でした。本アプリはその倒伏概念
（株＝Kabu／茎＝Ine／倒伏ゾーン）を継承しつつ、

- 直感的なリアルタイムUI
- 物理ベース（曲げモーメント vs 稈強度）への作り直し
- 降雨・窒素・土壌水分・生育ステージなど**多様な状況**への対応
- 製品としての見栄え・出力機能

を備えた**販売可能レベル**のアプリとして再構築したものです。
