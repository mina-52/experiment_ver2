# 稲倒伏シミュレーション

田んぼで稲が風によって倒れる様子（**倒伏**）をコンピュータ上でシミュレーションするツールです。

---

## このシステムでできること

```
 パラメータ入力           シミュレーション             出力
┌──────────────┐        ┌──────────────────┐       ┌──────────────────┐
│ ・茎の強さ   │        │  フロッキング    │       │ CSVデータ保存    │
│ ・風の強さ   │ ──▶   │  アルゴリズム    │ ──▶  │ （ゾーン別割合） │
│ ・株の間隔   │        │  × 風のフロー場  │       │                  │
│ ・稈長 / 茎数│        │  30×30 グリッド  │       │ リアルタイム表示 │
└──────────────┘        │  最大18,000本の稲│       │ （色分けで可視化）│
                         └──────────────────┘       └──────────────────┘
```

| 機能 | 内容 |
|---|---|
| 実験の自動繰り返し | 同じ条件で最大 **100回** 繰り返し実行して統計データを収集 |
| パラメータ自動変化 | 茎強度・風速などを **0.1刻み** で自動的に変えながら実験 |
| 大規模シミュレーション | 最大 **18,000本** の稲を同時シミュレーション |

## 環境構築

**必要なソフトウェアは「Processing」1つだけです（無料）。**

### 手順

**① Processing をダウンロード**

公式サイト https://processing.org/download にアクセスし、Windows版をダウンロードします。

**② 解凍して起動**

ダウンロードした ZIP を解凍し、`processing.exe` をダブルクリックします。
> インストール不要。解凍したフォルダのまま使えます。

**③ プロジェクトを開く**

Processing のメニューから **ファイル → 開く** を選択し、以下のファイルを開きます。

```
experiment_ver2/original/e1/s1v2/s1v2.pde
```

**④ 実行**

左上の **▶（実行）ボタン** を押すとシミュレーションが始まります。

> **注意：データ保存先フォルダについて**
> シミュレーション結果は `s1v2/` の一つ上にある `data/e1/must0.0/` 等に保存されます。
> フォルダが存在しないとエラーになるため、**実行前に対応するフォルダを作成**してください。

## システムの仕組み

### グリッドの全体像

田んぼ全体を **30 × 30 のマス目**に分割し、各マスに1株（Kabu）を配置します。

```
┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─ … ─┐
│●│●│●│●│●│●│●│●│●│●│   │  ● = 1株（Kabu）
├─┼─┼─┼─┼─┼─┼─┼─┼─┼─┤   │
│●│●│●│●│●│●│●│●│●│●│   │  各株の中に
├─┼─┼─┼─┼─┼─┼─┼─┼─┼─┤   │  稲20本が入っている
│●│●│●│●│●│●│●│●│●│●│   │
├─┼─┼─┼─┼─┼─┼─┼─┼─┼─┤   │  900株 × 20本
│ ︙ ︙ ︙ ︙ ︙ ︙ ︙ ︙ ︙ ︙ │   │  = 最大18,000本
└─┴─┴─┴─┴─┴─┴─┴─┴─┴─ … ─┘
         30 × 30 = 900株
```

### 倒伏ゾーン（5段階）

稲が**中心から離れるほど「倒れた」**と判断します。

```
      株の中心
         │
    ┌────┼────┐
    │ ┌──┼──┐ │
    │ │ ┌┼┐ │ │
    │ │ │●│ │ │  ← Zone 1（直立）
    │ │ └┘ │ │  ← Zone 2（少し傾いた）
    │ └────┘ │  ← Zone 3（半分倒れた）
    └────────┘  ← Zone 4（ほぼ倒れた）
                ← Zone 5（完全に倒れた）

   中心に近い = 直立    外側に遠い = 倒伏
```

| Zone | 状態 | 画面の色 |
|---|---|---|
| Zone 1 | 直立 | 青 |
| Zone 2 | わずかに傾いた | 青緑 |
| Zone 3 | 半分倒れた | 緑 |
| Zone 4 | ほぼ倒れた | 黄 |
| Zone 5 | 完全に倒れた | オレンジ |

> **ルール：** 一度外側のゾーンに移ったら元には戻りません（倒れた稲は元に戻らない）。

**終了条件：** Zone1 + Zone2 の合計が **50% 以下** になったとき、または **18,000フレーム** 超過したとき。

### 風のサイクル

無風と突風が交互に繰り返されます。

```
風速
 強│         ████         ████
   │         ████         ████
 弱│ ─────── ████ ─────── ████ ──…
   └──────────────────────────────▶ 時間（フレーム）
      無風       突風   無風    突風
     75フレーム  50F  75フレーム 50F
     ├──────────────────┤
           1サイクル（125フレーム）
```

### フロッキングアルゴリズム（群れの動き）

各稲は3つのルールに従って動きます。鳥の群れや魚の群れと同じ仕組みです。

```
① 分離（Separation）  ② 整列（Alignment）  ③ 凝集（Cohesion）
   重み: 0.6（最強）      重み: 0.1（最弱）     重み: 0.3（中）

   ↖  ↑  ↗             →  →  →            →  ↘  →
   ← [●] →             →  →  →            ↗ [中] ↙
   ↙  ↓  ↘             →  →  →            →  ↗  →

   近い稲と           近くの稲と          近くの稲の
   離れようとする     同じ方向に動く      中心に集まる
```

## プログラムの流れ

```
[起動（setup）]
      │
      ▼
[試行開始（startTrial）]── CSVファイル作成・初期化
      │
      ▼
┌─────────────────────────────────────┐
│  毎フレーム繰り返し（draw）          │
│                                     │
│  1. 風の強さを更新（無風 or 突風）  │
│  2. 全Kabu・Ineの動きを計算         │
│  3. ゾーン別の割合を集計            │
│  4. 結果をCSVに書き込み             │
│                                     │
│  ┌─ 終了条件を満たした？ ──────────┘
│  │       Yes
└──┘
      │
      ▼
[次の試行へ（nextTrial）]── 100回まで繰り返す
      │  100回完了
      ▼
[パラメータを0.1増やして次の実験]
      │  全実験完了
      ▼
[プログラム終了]
```

## 実験パラメータ

`s1v2.pde` の先頭にある変数を変えることで異なる条件で実験できます。

| 変数名 | 意味 | デフォルト値 | 説明 |
|---|---|---|---|
| `must` | 茎強度 | `0.0` | 稲の茎の硬さ。大きいほど風に耐えやすい（0〜10） |
| `kan` | 稈長 | `120` | 稲の茎の長さ（px）。長いほど倒れやすい |
| `interval` | 株間隔 | `20` | 株と株の間隔（px） |
| `m` | 茎数 | `20` | 1株あたりの稲の本数 |
| `ws1` | 突風時の風速 | `1.0` | 突風時の風の強さ |
| `startzone` | 初期倒伏ゾーン | `1` | シミュレーション開始時のゾーン |
| `pattern` | 変化させるパラメータ | `"must"` | 実験で自動変化させる変数の名前 |
| `step` | 変化幅 | `0.1` | 実験ごとにパラメータを増やす量 |

### 変更例

```java
// 茎強度を 5.0 に変える
float must = 5.0;

// 稈長を変化させる実験に変える
String pattern = "kan";
```

`pattern` に設定できる値：

| 値 | 変化させるパラメータ |
|---|---|
| `"must"` | 茎強度 |
| `"kan"` | 稈長 |
| `"iv"` | 株間隔 |
| `"ws"` | 風速 |
| `"m"` | 茎数 |
| `"sz"` | 初期倒伏ゾーン |

## 出力CSVデータの説明

### ファイル名の形式

```
e1  _  must0.0  _  x1  _  20260313_1436  .csv
│       │          │        │
│       │          │        └── 日時（年月日_時分）
│       │          └────────── 試行番号
│       └───────────────────── パラメータと値
└───────────────────────────── 実験番号
```

### CSVの内容

```
frame , zone1_ratio , zone2_ratio , zone3_ratio , zone4_ratio , zone5_ratio
  1   ,    0.860    ,    0.139    ,    0.000    ,    0.000    ,    0.000
  50  ,    0.710    ,    0.180    ,    0.090    ,    0.015    ,    0.005
 240  ,    0.300    ,    0.190    ,    0.200    ,    0.180    ,    0.130
```

- **1行 = 1フレーム分**のデータ
- 各ゾーンにいる稲の**割合（0.0〜1.0）** が記録される
- 各行の合計は必ず `1.0`（zone1 + zone2 + zone3 + zone4 + zone5 = 1.0）

**読み方の例：**
`frame=240, zone1_ratio=0.30` → 240フレーム時点で全稲の 30% しか直立していない（= 70% が倒れた）

## キーボード操作

実行中にキーを押すと表示を切り替えられます。

| キー | 効果 |
|---|---|
| `1` | 境界円（ゾーンの範囲）の表示 ON/OFF |
| `2` | 株の中心への線の表示 ON/OFF |
| `3` | 穂（稲の粒）の表示 ON/OFF |

## よくあるエラーと対処法

| 症状 | 原因 | 対処法 |
|---|---|---|
| CSVが保存されない | `data/` フォルダが存在しない | `data/e1/must0.0/` などのフォルダを手動で作成する |
| すぐに終了する | 終了条件を満たした正常な動作 | コンソールに `All trials completed` と出ていれば正常 |
| 画面サイズがおかしい | パラメータから自動計算される仕様 | `kan`・`interval` の値を確認する |

## ソースファイルの役割

| ファイル名 | 役割 |
|---|---|
| `s1v2.pde` | メインプログラム。実験設定・描画ループ・CSV保存を担当 |
| `Ine_v2.pde` | 稲1本の動きを定義するクラス（フロッキング・物理演算） |
| `Kabu_v2.pde` | 複数の稲をまとめた「株」クラス（株間の相互影響も担当） |
| `Flowfield_v2.pde` | Perlin ノイズで風の方向を表すフロー場クラス |
